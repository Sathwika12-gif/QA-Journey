Software Testing Life Cycle (STLC)

Requirement Analysis → Test Planning → Test Case Design → Test Execution → Test Closure

Bug Life Cycle

New → Assigned → Open → Fixed → Retest → Closed
1. What is manual testing?
Manual testing is the process of testing software manually without using automation tools. Testers execute test cases, observe application behavior, and identify defects.
2. What is software testing?
Software testing verifies and validates that an application works according to requirements and is free from critical defects.
3. What is verification?
Verification checks whether the software is built according to design specifications without executing the application.
4. What is validation?
Validation checks whether the software meets user expectations by executing the application.
5. What is a test case?
A test case is a document containing test steps, test data, expected results, and preconditions to verify functionality.
6. What is a test scenario?
A test scenario is a high-level description of what functionality needs to be tested.
7. What is a bug?
A bug is an error in software that causes unexpected or incorrect behavior.
8. What is a defect?
A defect is the difference between expected and actual application behavior.
9. What is severity?
Severity indicates the impact of a defect on system functionality.
10. What is priority?
Priority indicates how quickly a defect should be fixed.
11. What is regression testing?
Regression testing ensures that previously working features continue to work after code changes.
12. What is smoke testing?
Smoke testing checks basic functionality to ensure the build is stable enough for further testing.
13. What is sanity testing?
Sanity testing verifies specific functionality after minor changes or bug fixes.
14. What is STLC?
STLC stands for Software Testing Life Cycle which includes requirement analysis, planning, test design, execution and closure.
15. What is SDLC?
SDLC stands for Software Development Life Cycle which includes requirement gathering, design, development, testing and deployment.
16. What is black box testing?
Black box testing verifies functionality without knowledge of internal code.
17. What is white box testing?
White box testing verifies internal code structure and logic.
18. What is functional testing?
Functional testing checks application features against business requirements.
19. What is non-functional testing?
Non-functional testing verifies performance, security, usability and reliability.
20. What is exploratory testing?
Exploratory testing is an approach where testers simultaneously learn the application and execute tests.
21. What is UAT?
User Acceptance Testing verifies whether the system satisfies business requirements.
22. What is alpha testing?
Alpha testing is internal testing performed before releasing the product to external users.
23. What is beta testing?
Beta testing is performed by real users to collect feedback before final release.
24. What is boundary value analysis?
Boundary Value Analysis tests edge values where defects are most likely to occur.
25. What is equivalence partitioning?
Equivalence partitioning divides input data into valid and invalid groups to reduce test cases.
26. What is defect leakage?
Defect leakage occurs when defects are missed during testing and discovered later in production.
27. What is defect density?
Defect density measures the number of defects per module or lines of code.
28. What is test plan?
A test plan describes testing scope, strategy, resources, schedule and deliverables.
29. What is test strategy?
Test strategy defines the high-level testing approach followed by the project.
30. What is test data?
Test data is the input data used to execute test cases.
31. What is a build?
A build is a compiled version of the application delivered by developers for testing.
32. What is release?
A release is the final version delivered to customers after testing.
33. What is compatibility testing?
Compatibility testing ensures the application works across browsers, devices and operating systems.
34. What is usability testing?
Usability testing evaluates how user-friendly an application is.
35. What is risk-based testing?
Risk-based testing prioritizes testing based on business risk and critical features.
36. What is ad-hoc testing?
Ad-hoc testing is informal testing without predefined test cases.
37. What is positive testing?
Positive testing verifies application behavior with valid inputs.
38. What is negative testing?
Negative testing verifies application behavior with invalid inputs.
39. What is test environment?
Test environment includes hardware, software, tools and configurations used for testing.
40. What is bug report?
A bug report contains bug description, steps to reproduce, expected result, actual result and screenshots.
41. What is retesting?
Retesting verifies that a previously failed test case passes after a defect is fixed.
42. Difference between retesting and regression testing?
Retesting focuses on verifying a specific defect fix while regression testing verifies that other functionality still works.
43. What is test suite?
A test suite is a collection of related test cases.
44. What is test harness?
Test harness is a set of tools and scripts used for executing tests.
45. What is agile testing?
Agile testing is testing performed continuously during agile development cycles.
46. What is sprint?
A sprint is a short development cycle where features are developed and tested.
47. Role of QA in agile?
QA collaborates with developers, writes test cases early, and performs continuous testing.
48. What is RTM?
Requirement Traceability Matrix links requirements with test cases to ensure full coverage.
49. What is defect lifecycle?
Defect lifecycle describes stages from defect discovery to closure.
50. What is acceptance testing?
Acceptance testing ensures the system meets business requirements before release.
51. How would you test a login page?
Test valid login, invalid credentials, empty fields, password masking and security validations.
52. How would you test a signup form?
Verify field validation, email format, password rules and duplicate user scenarios.
53. How would you test a search feature?
Verify valid keywords, invalid inputs, partial matches and performance of search results.
54. How would you test a checkout process?
Test product selection, cart update, payment gateway, order confirmation and email notification.
55. How would you test file upload functionality?
Verify file formats, file size limits and upload errors.
56. How would you test password reset?
Verify reset email, token validity and password rules.
57. What if requirements are unclear?
Clarify with product owners or developers and document assumptions.
58. What if a bug is not reproducible?
Verify environment, logs and provide detailed steps to developers.
59. What if build is unstable?
Perform smoke testing and report critical failures.
60. How do you handle tight deadlines?
Prioritize high-risk features and perform risk-based testing.
61. What is database testing?
Database testing verifies the data stored in the database, including tables, relationships, triggers, and stored procedures. Testers validate that data is correctly saved, updated, and retrieved by the application.
62. What is data integrity testing?
Data integrity testing ensures that data remains accurate, consistent, and reliable throughout the system. It verifies database constraints, relationships, and data correctness.
63. What is API testing?
API testing verifies the functionality, reliability, performance, and security of APIs. Testers validate request responses, status codes, and data accuracy between systems.
64. What are common HTTP status codes in API testing?
Common status codes include 200 (success), 201 (created), 204 (no content), 400 (bad request), 401 (unauthorized), 403 (forbidden), 404 (not found), and 500 (server error).
65. What is test coverage?
Test coverage measures how much of the application functionality or code is covered by test cases.
66. What is a defect tracking tool?
A defect tracking tool is used to record, track, and manage bugs during the testing lifecycle. Examples include Jira, Bugzilla, and Azure DevOps.
67. What information should be included in a bug report?
A bug report should include bug ID, summary, description, steps to reproduce, expected result, actual result, screenshots, environment details, and severity.
68. What is production environment?
Production is the live environment where real users interact with the application after deployment.
69. What is staging environment?
Staging is a pre-production environment that closely mirrors the production setup and is used for final testing before release.
70. What is test closure?
Test closure is the final phase of STLC where test results are documented, test metrics are analyzed, and lessons learned are recorded.
71. What is test metric?
Test metrics are measurements used to evaluate testing progress and quality such as test coverage, defect density, and pass/fail rate.
72. What is defect prevention?
Defect prevention focuses on identifying root causes of defects and improving development and testing processes to prevent future issues.
73. What is test execution?
Test execution is the process of running test cases and comparing expected results with actual results to identify defects.
74. What is test log?
A test log records details of executed test cases including execution time, tester name, and results.
75. What is a blocker defect?
A blocker defect prevents further testing of the application and must be fixed immediately.
76. What is a critical defect?
A critical defect affects major functionality of the system but may still allow limited testing to continue.
77. What is minor defect?
A minor defect has little impact on functionality and usually affects cosmetic or non-critical features.
78. What is GUI testing?
GUI testing verifies the graphical user interface including buttons, layouts, colors, fonts, and responsiveness.
79. What is localization testing?
Localization testing verifies that the application works correctly in different languages and regional settings.
80. What is internationalization testing?
Internationalization testing ensures that the application can support multiple languages and cultural settings.
81. What is configuration testing?
Configuration testing verifies application behavior across different hardware, operating systems, browsers, and networks.
82. What is installation testing?
Installation testing verifies that the application installs correctly and runs properly after installation.
83. What is recovery testing?
Recovery testing verifies how well the system recovers from crashes, hardware failures, or other unexpected problems.
84. What is load testing?
Load testing evaluates how the application behaves under expected user load conditions.
85. What is stress testing?
Stress testing determines how the application behaves under extreme load conditions beyond its limits.
86. What is performance testing?
Performance testing measures system responsiveness, stability, and speed under different workloads.
87. What is security testing?
Security testing identifies vulnerabilities in the system and ensures data protection from unauthorized access.
88. What is penetration testing?
Penetration testing simulates cyber attacks to evaluate system security vulnerabilities.
91. What is cache testing?
Session testing verifies session management such as login sessions, timeout behavior, and session security.
90. What is cookie testing?
Cookie testing verifies how cookies store and manage user information within the browser.
91. What is cache testing?
Cache testing verifies whether the application correctly stores and retrieves cached data for performance improvement.
92. What is URL testing?
URL testing verifies proper redirection, valid links, and handling of invalid URLs.
93. What is accessibility testing?
Accessibility testing ensures that applications are usable by people with disabilities, such as screen reader compatibility.
94. What is compatibility testing?
Compatibility testing verifies the application works across different browsers, devices, and operating systems.
95. What is mobile testing?
Mobile testing verifies mobile applications on different devices, screen sizes, and operating systems.
96. What is cloud testing?
Cloud testing verifies applications hosted in cloud environments such as AWS, Azure, or Google Cloud.
97. What is test automation?
Test automation uses tools like Selenium, Cypress, or Playwright to execute test cases automatically.
98. What is continuous testing?
Continuous testing integrates automated tests into CI/CD pipelines to ensure quality during development.
99. What is CI/CD?
CI/CD stands for Continuous Integration and Continuous Delivery which automates build, testing, and deployment processes.
100. What is the role of a QA engineer?
A QA engineer ensures software quality by designing tests, identifying defects, collaborating with developers, and validating that applications meet user requirements.
