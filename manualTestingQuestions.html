Manual Testing Interview Questions

Manual testing is a fundamental skill for every QA engineer and software tester. This guide contains the most commonly asked manual testing interview questions with answers to help both beginners and experienced testers prepare for interviews.

These questions are based on real-world QA scenarios used in Agile development environments.

Topics Covered

  • Verification vs Validation
  • Test Scenario vs Test Case
  • Bug Life Cycle
  • Sanity Testing
  • Regression Testing
  • STLC
  • Exploratory Testing
1. What is verification vs validation?

Verification checks whether the product is built according to specifications without executing the code, while validation checks whether the product meets user expectations by executing the application.

2. What is a test scenario?

A test scenario is a high-level description of what functionality needs to be tested.

3. What is a test case?

A test case is a detailed set of steps including test data, expected results, and execution conditions.

4. Explain the bug life cycle.

New → Assigned → Open → Fixed → Retesting → Closed.

5. What is sanity testing?

Sanity testing verifies specific functionality after a bug fix or small change.

6. What is severity vs priority?

Severity defines the impact of a bug while priority defines how quickly it should be fixed.

7. What is regression testing?

Regression testing ensures that existing functionality works correctly after code changes.

8. Smoke vs sanity testing

Smoke testing checks major functionalities of a build while sanity testing checks specific changes.

9. How do you write a bug report?

Include bug ID, description, steps to reproduce, expected result, actual result, and screenshots.

10. What if requirements are unclear?

Discuss with the BA or developers and perform exploratory testing.

11. Black box testing techniques

Equivalence Partitioning, Boundary Value Analysis, Decision Table Testing, State Transition Testing.

12. Test case design techniques

Equivalence Partitioning, Boundary Value Analysis, Decision Table Testing, State Transition Testing.

13. Types of non-functional testing

Performance, Load, Stress, Security, Usability, Compatibility testing.

14. STLC stages

Requirement Analysis → Test Planning → Test Case Design → Test Execution → Test Closure.

15. Entry vs Exit criteria

Entry criteria defines when testing starts while exit criteria defines when testing ends.

16. What is exploratory testing?

Exploratory testing involves learning and executing tests simultaneously without predefined test cases.

17. Functional vs non-functional testing

Functional testing verifies features while non-functional testing verifies performance and usability.

18. What is UAT?

User Acceptance Testing verifies whether the system meets business requirements.

19. Retesting vs regression testing

Retesting verifies bug fixes while regression testing ensures existing functionality still works.

20. What is a test plan?

A document describing testing scope, strategy, resources, and schedule.

21. Test case vs test script

A test case is manual testing documentation while a test script is used for automation.

22. What is defect leakage?

Defect leakage occurs when defects are missed in testing and found in production.

23. What is defect density?

Defect density measures the number of defects per module or lines of code.

24. What is a test summary report?

A document summarizing executed test cases, results, and defects.

25. What are test environments?

Development, QA/Test, Pre-production, and Production environments.
