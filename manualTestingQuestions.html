Manual Testing Interview Questions

Manual testing is a fundamental skill for every QA engineer and software tester. This guide contains the most commonly asked manual testing interview questions with answers to help both beginners and experienced testers prepare for interviews.

These questions are based on real-world QA scenarios used in Agile development environments.

Topics Covered

  • Verification vs Validation
  • Test Scenario vs Test Case
  • Bug Life Cycle
  • Sanity Testing
  • Regression Testing
  • Black Box Testing Techniques
  • STLC
  • Exploratory Testing
1. What is the difference between verification and validation?

Verification checks whether software is built according to requirements without executing code. Validation checks whether the application meets user expectations by executing the application.

2. What is a test scenario and how is it different from a test case?

A test scenario is a high-level description of what needs to be tested. A test case is a detailed step-by-step instruction including test data and expected results.

3. Explain the bug life cycle.

New → Assigned → Open → Fixed → Retesting → Closed. If the issue persists it can be reopened.

4. What is sanity testing?

Sanity testing verifies that specific functionality works correctly after a bug fix or small change.

5. What will you do if a developer says "This is not a bug"?

Re-check the requirement, provide evidence, and discuss with the BA or product owner if needed.

6. Difference between severity and priority

Severity measures the impact of a defect while priority determines how quickly it should be fixed.

7. What is regression testing?

Regression testing ensures existing features work after code changes or bug fixes.

8. Difference between smoke testing and sanity testing

Smoke testing verifies major functionality of a build while sanity testing checks specific functionality after changes.

9. How do you write a good bug report?

Include bug ID, title, description, steps to reproduce, expected result, actual result, environment details, and screenshots.

10. What if requirements are unclear?

Discuss with BA, product owner, or developers and perform exploratory testing.

11. Black box testing techniques

Equivalence Partitioning, Boundary Value Analysis, Decision Table Testing, State Transition Testing.

12. Test case design techniques

Equivalence Partitioning, Boundary Value Analysis, Decision Table Testing, State Transition Testing.

13. Types of non-functional testing

Performance Testing, Load Testing, Stress Testing, Security Testing, Usability Testing, Compatibility Testing.

14. STLC stages

Requirement Analysis, Test Planning, Test Case Design, Test Environment Setup, Test Execution, Test Closure.

15. Entry criteria and exit criteria

Entry criteria define when testing begins while exit criteria define when testing stops.

16. What is exploratory testing?

Exploratory testing involves learning, designing, and executing tests simultaneously.

17. Functional vs non-functional testing

Functional testing checks system features while non-functional testing checks performance and usability.

18. What is UAT?

User Acceptance Testing verifies the application meets business requirements before release.

19. Retesting vs regression testing

Retesting verifies bug fixes while regression testing ensures existing functionality still works.

20. What is a test plan?

A document describing testing scope, strategy, schedule, resources, and environment.

21. Test case vs test script

A test case is used for manual testing while a test script is used for automation.

22. What is defect leakage?

Defect leakage occurs when bugs are missed during testing and found in production.

23. What is defect density?

Defect density measures the number of defects per module or per lines of code.

24. What is a test summary report?

A document summarizing testing results, executed test cases, and defects found.

25. Test environments

Development, QA/Test, Pre-production, and Production environments.
